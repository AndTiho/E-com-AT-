# Проект "E-com(AT)" 
## Описание
Учебный проект по созданию приложения для электронной торговли.\
**В данный момент находится в разработке, просим проявить терпение.**
## Установка:
1. Клонируйте репозиторий:
```
git@github.com:AndTiho/E-com-AT-.git
```
2. Установите зависимости:
```
poetry install
```
## Использование
1. В модуле [Категория](src\category.py):\
Определён класс Категория, в котором определены следующие свойства:
* название (name)
* описание (description)
* список товаров категории (products)

Реализован метод:
* Для добавления нового продукта в категорию, если он является классом Продукты или его наследником.

2. В модуле [Продукт](src\product.py):\
Определён класс Продукт, в котором определены следующие свойства:
* название (name)
* описание (description)
* цена (price)
* количество в наличии (quantity)

Реализованы методы:
* Для сложения двух продуктов в виде:\
(цена запрашиваемого товара * количество) + (цена другого товара * количество)
* Для создания объекта класса Product из словаря.
Если такой продукт уже существует, то добавляет к нему новое количество товара
и если новая цена выше старой, то меняет её в большую сторону.
* Сеттер, который принимает на вход новую цену товара и выполняет проверку на корректность.\
Если цена ниже имеющейся, то запрашивает у USER команду на замену цены

3. В модуле [Утилиты](src\utils.py) находятся:
* Функция, которая принимает на вход путь к файлу с данными JSON
и возвращает данные для работы в Python
* Функция, которая преобразует словарь в объекты заявленных выше классов.

4. В модуле [Итератор](src\prod_iterator.py):\
Определён вспомогательный класс, с помощью которого можно перебирать товары одной категории,
например в цикле for.

5. В модуле [Смартфон](src\smartphone.py):\
Определён класс Смартфон, от родительского класса Продукт, в котором определены дополнительные свойства:
* эффективность (efficiency)
* модель (model)
* память (memory)
* цвет (color)

Реализован метод:
* Для сложения двух продуктов в виде:\
(цена запрашиваемого товара * количество) + (цена другого товара * количество), если они оба являются классом Смартфоны.

6. В модуле [Трава газонная](src\lawn_grass.py):\
Определён класс Трава газонная, от родительского класса Продукт, в котором определены дополнительные свойства:
* страна производитель (country)
* срок прорастания (germination_period)
* цвет (color)

Реализован метод:
* Для сложения двух продуктов в виде:\
(цена запрашиваемого товара * количество) + (цена другого товара * количество), если они оба являются классом Трава газонная.

7. В модуле [Миксин для печати](src\print_mixin.py):\
Определён миксин для отображения в консоли записи о том, как формировался объект класса

8. В модуле [Базовый продукт](src\base_product.py):\
Определён базовый класс для всех видов Продуктов которые инициироваться в процессе добавления товаров на склад.

9. В модуле [Заказ](src\order.py):\
Определён класс для формирования заказов, включающие в себя такие свойства:
* наименование товара (Product)
* количество (quantity)
* приобретаемое свойство "общая стоимость" (total)

В данный момент, для корректной проверки работы каждой функций, вы можете воспользоваться
оператором `print` и ввести все необходимые данные, соответствующие требуемым аргументам функций.

## Запуск и ожидаемое поведение программы
Для запуска программы воспользуйтесь:
```
python main.py
```
Программа находится на стадии разработки.\
Для работы программы прилагаются данные транзакций в формате
JSON, которые можно использовать в качестве тестирования работы функции.

## Тестирование:
Наш проект покрыт юнит-тестами Pytest. Для их запуска выполните команду:

`pytest`
